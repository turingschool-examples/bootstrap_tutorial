Turing M2 Bootstrap Tutorial

Goals

By the end of this tutorial, you will know/be able to:

  • Have a basic understanding of Bootstrap and its capabilities.
  • Learn how adding Bootstrap classes to an element can change its style.
  • Learn how to use the Bootstrap documentation to solve issues.
  • Understand what responsive web design is.
  • Understand the container.
  • Understand the grid system of CSS Frameworks.
  • Understand how to use glyphicons.
  • Be able to translate this knowledge to other CSS Frameworks including, but not limited to Skeleton, Materialize and Foundation.

Review

  • HTML is a markup language used to describe content.
  • CSS can be used to style that content/change the way that it is displayed on a page.
  • JavaScript can be used to further enhance both the look and behavior of our websites.

At a high level, content itself is made up of words, images, videos, etc., but is not in and of itself very different from other documents that people produce. HTML, CSS, and JavaScript together help shape the way that our browsers display that content.

Background

Bootstrap is a CSS/JavaScript framework. At its core it looks to provide developers with a starter kit for the CSS and JavaScript that they might want to use on their pages. Many developers have written code to produce a navbar with dropdowns; Bootstrap makes it so that you don't have to reinvent the wheel each time you want that same functionality.

  • Click this link to get a sense of how much CSS was used to style this page. Not very much, right?
  • Click this link to see how much CSS is included when you use Bootstrap in your project. A ton!

Using Bootstrap is using styles that other people wrote so that you don't have to. It will continue to be important to have a fundamental understanding of how HTML/CSS/JavaScript work together to make a webpage work, but Bootstrap will allow you to create a decent looking page without too much initial setup.

Basic Three Column Layout

One of the goals of this tutorial is to begin to get you familiar with reading documentation for the tools that you use. Before you dig in, take a moment and look at the Bootstrap website. In particular, look at the Getting Started, CSS, and Components sections of the site.

Once you've had a little bit of an opportunity to dig around, go back to the Getting Started page and download and unzip Bootstrap.

Getting Set Up

  • In the directory that you forked for this tutorial, create a new three_column directory.
  • Find the basic template on the Getting Started page, and put it into a new index.html file in our project directory. Notice that in that file you have the following lines:
    • A link to css/bootstrap.min.css
    • A script tag to a JavaScript library (jQuery) hosted by Google
    • A script tag pointing to a local copy of bootstrap.min.js in a js folder
  • Open your index.html page in your browser, and you should have a pretty blandly styled "Hello World." Note: you don't need to host these pages online or anything like that. You can simply open the local file in your browser (Chrome, Firefox, Safari, etc.). Most of the time you should be able to use a command-o shortcut or find Open File... in the File menu.
  • If you press ctrl-opt-i and look at the console (assuming you're using Chrome) you should see that the CSS and JavaScript files that are referenced in this page aren't yet loading.
  • In order for them to load, you'll need to move the js, css, and fonts folders that you downloaded from the Bootstrap site to the new three_column directory so that your new index.html file references them appropriately. Do that now.
  • Open your index.html file in the browser again to see what you have. Ideally, you'll still see "Hello World," but now it will be styled in a sans-serif font.
  • Celebrate!

Adding Content

Sometimes you just need a little bit of content to see what your page will look like with words on it. When that happens, lots of people use lorem ipsum, or some other dummy text. If you're in a professional setting, the classic version is usually a good choice, but if you're working on something for yourself or friends there are lots of other choices.

  • Find three paragraphs of lorem ipsum and add it to the body of your index.html file.
  • Add titles above each of the paragraphs.
  • Wrap the paragraphs in p tags and the headers in h2 tags.
  • Open the file in your browser again to see what you have now. There should be three paragraphs with heading stacked on top of each other.

Using Bootstrap

Now for the new stuff. We're going to use some of the styles that Bootstrap gives us to style those paragraphs so that instead of showing up on top of each other they line up side by side. One of the things you'll notice is that styling with Bootstrap often becomes an exercise in trying to figure out how some of the code snippets they offer you can be used on your content. To that end:

  • Go back to the CSS section of the Bootstrap site to see what we can find on grids.
  • On that page, there should be an example called Stacked-to-horizontal. Copy the code from that example and put it into your index.html file below the paragraphs that you have already added.
  • If you open your page now, you should see some text like .col-md-1 peppering the bottom of your screen. It might be hard to tell, but those snippets are arranged into a grid made up of twelve columns. Elements styled with a .col-md-1 class have a width 1/12 the size of the total, while elements styled with a .col-md-6 class have a width 1/2 the size of the total.
  • If you open the dev tools again (cmd-opt-i in Chrome) and click on the small box with an arrow in the top left corner of the console (like this ), you should be able to hover over different snippets of text to see the box that contains them.
  • As you do that, notice that the three column grid that we want is represented by the divs styled with a class of .col-md-4. Replace the .col-md-4 snippets with our headers and paragraphs
  • If you open the page now, you should see that we're getting close. We just have the other text snippets still hanging out. Delete those rows from our index.html and we should have the three column view that we were looking to create.
  • The one big issue I see remaining is that our layout seems to be bumping right up against the edge of the page. Sometimes we want this behavior (look at the headings in this document for example), but in this case we most likely want to give the content some breathing room. In your index.html file, put a div tag before the Hello World h1 and add a closing tag after the last div associated with your newly created columns. Give the first div a class of container (e.g. <div class="container">). That should push your content to the center.
  • If you'd like, at this point you can create an additional stylesheet in the css folder and add some custom styles to further change the look of these columns (e.g. add a background-color, or border). Be sure to include the link to the new stylesheet underneath the link to the Bootstrap stylesheet in the head of your document.

Navbar

Let's say we also want to add a navbar to the page that we're working on. We'll follow a similar process. Check to see if you can find the code you need on the Components section of the Bootstrap site. Once you've done that, copy it to your index file, and make adjustments until you have it looking the way you'd like.

Be sure to check on your progress along the way. With Bootstrap, you might run into a situation where you've removed something without fully understanding what it does. To prevent this, be sure to check in frequently to see that you still have the functionality you expect as you make changes.

Spend some additional time on the Components page to get a sense for what Bootstrap has to offer.
